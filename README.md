# 📦 Cloud-Sync Inventory Management System (React)

A professional, modern inventory solution built for warehouse operations and stock management. This application provides real-time cloud syncing, project-based stock reservations, and a full transaction audit trail.
Built with React, Vite, Tailwind CSS, and powered by Supabase.

> ✨ **Vibe Coded**: This application was created using **Vibe Coding**—leveraging advanced AI models (such as Gemini for database/core logic and Claude for frontend UI generation) to rapidly build, debug, and iterate on the software architecture.

---

## ✨ Features
* **Real-time Cloud Sync**: Powered by Supabase—access your inventory instantly.
* **Add & Edit Products**: Manage your catalog directly from the UI with dedicated modals.
* **Bulk In/Out Operations**: Rapidly update stock levels for multiple items at once via the Bulk tools.
* **Project Allocations**: Reserve items for specific projects and track movements (IN/OUT/RETURN).
* **ProjectCode Search**: Quickly track down transactions by their associated Project Code, Remark, or PO Number.
* **Data Portability**: Export your Stock, Bulk Operations, and Transactions directly to CSV files.
* **Robust UI**: Dark-themed, responsive interface built with Tailwind CSS.

---

## 🛠 Tech Stack
* **Frontend Framework**: React 18, utilizing Hooks and Context API for state management.
* **Build Tool**: Vite for lightning-fast development and bundling.
* **Styling**: Tailwind CSS for utility-first styling.

## � Getting Started

### 1. Database Setup (Supabase)

You must initialize your database in Supabase and configure **Row Level Security (RLS)**.
Run the following SQL in your [Supabase SQL Editor](https://supabase.com/dashboard/project/_/sql):

```sql
-- Products Table
CREATE TABLE IF NOT EXISTS products (
  "modelNumber" TEXT PRIMARY KEY,
  "suffix" TEXT,
  category TEXT,
  "currentQuantity" INT DEFAULT 0,
  location TEXT,
  barcode TEXT,
  description TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Transactions Table
CREATE TABLE IF NOT EXISTS transactions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  "productId" TEXT REFERENCES products("modelNumber") ON DELETE CASCADE,
  "displayName" TEXT,
  "stock_change" INT,
  "stockAfter" INT,
  project TEXT,
  "projectAction" TEXT,
  "poNumber" TEXT,
  reason TEXT
);

-- Settings Table (Stores Categories & Project Allocations)
CREATE TABLE IF NOT EXISTS settings (
  key TEXT PRIMARY KEY, 
  value JSONB DEFAULT '[]'::jsonb,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Security (RLS)
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE settings ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Enable all access for anon" ON products FOR ALL TO anon USING (true) WITH CHECK (true);
CREATE POLICY "Enable all access for anon" ON transactions FOR ALL TO anon USING (true) WITH CHECK (true);
CREATE POLICY "Enable all access for anon" ON settings FOR ALL TO anon USING (true) WITH CHECK (true);

GRANT ALL ON ALL TABLES IN SCHEMA public TO anon;
GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO anon;
```

### 2. Local Environment Setup

Clone the repository and install dependencies:

```bash
npm install
```

Create a `.env` file in the root directory and add your Supabase credentials:

```env
VITE_SUPABASE_URL=https://your-project-url.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

*Note: If these aren't provided in the `.env` file, the app will fall back to prompting for them on first load and storing them in `localStorage`.*

### 3. Run Development Server

```bash
npm run dev
```

The application will be available at `http://localhost:5173` (or the port specified by Vite).

## � Usage Notes & Workflows

> [!WARNING]
> **Stock Modification Restrictions**
> To maintain strict auditing, the **Quantity** field is locked when editing or adding a product. All physical stock modifications **MUST** be performed via the **Bulk In Stock** or **Bulk Out Stock** tools so that a permanent transaction record is generated.

> [!TIP]
> **Navigating to Projects**
> From the main "Stock" tab, clicking the **Project...** button on any table row will immediately jump you to the Projects Tab with that specific product pre-selected, allowing you to instantly allocate or return stock for that item.

> [!IMPORTANT]
> **Search Fallbacks**
> If a product is completely deleted from the database but retains history in the Transactions tab, attempting to search for it using the ProjectCode scanner will securely fall back to displaying its originally logged `displayName`.
